# CSV Generation Tool Technical Context

## Overview
We will implement a new tool `csv_generation_tool` for the chat agent. This tool serves as a "data formatter" or "processor" in a pipeline. It takes data provided by previous tool calls (like `metadata_search` or `read_contracts`) and converts it into a structured CSV file based on user instructions. The generated CSV is returned directly in the chat response.

## Architecture
We will create a new directory `app/services/chatbot/tools/csv_generation/`.

### Directory Structure
```
app/services/chatbot/tools/csv_generation/
├── __init__.py
├── csv_generation_tool.py      # Main entry point, defines the FunctionTool
├── generate_csv.py             # Core logic: LLM processing, CSV creation
└── prompts.py                  # Prompt templates for the LLM
```

## Implementation Details

### 1. `csv_generation_tool.py`
This file will define `get_csv_generation_tool()`.
It will configure the `FunctionTool` with:
- **Name**: `csv_generation_tool`
- **Description**: Instructions to the Agent.
    - "Use this tool to generate a CSV file from data you have already validated or fetched."
    - "Do NOT use this tool to find contracts. Use `metadata_search`, `semantic_search`, or `read_contracts` first."
    - **Sources:** `metadata_search`, `semantic_search`, `read_contracts` or `get_file_content_tool`
    - **Logic:**
        1. Agent receives request (e.g., "Export these contracts/files to CSV").
        2. Agent identifies necessary data source and calls appropriate tool first.
        3. Agent passes result (List[Dict] or Dict) + User Instruction to `csv_generation_tool`.
- **Parameters**:
    - `data` (`List[Dict[str, Any]]`): The input data to process. This should be the list of contracts or metadata retrieved from a previous tool call (e.g., `metadata_search`, `semantic_search`, `read_contracts`, or `get_file_content_tool`).
    - `instruction` (`str`): The user's specific instruction on how to frame the CSV (e.g., "compare key differences", "list columns for ID and Name").

### 2. `generate_csv.py`
This file will contain the async function `generate_csv_logic`.
**Workflow**:
1.  **Validation**: Ensure `data` is not empty.
2.  **LLM Processing**:
    - Construct a prompt using `prompts.py`.
    - Input: The `data` (context) and the `instruction`.
    - Task: Ask the LLM to process this data and output a structured JSON representing the CSV table (headers and rows).
    - Use `OpenAIResponses` with a Pydantic model (`CSVData` with `headers` and `rows`) to ensure valid structure.
3.  **CSV Creation**:
    - Convert the structured JSON to a CSV string using `csv.DictWriter` or `csv.writer`.
4.  **Response Construction**:
    - **Do NOT upload to GCS.**
    - Return a dictionary:
    ```python
    {
        "success": True,
        "csv_content": "Column1,Column2\nValue1,Value2...",
        "filename": "generated_data_dynamic_name.csv", # Generated by LLM based on user request
        "message": "CSV generated successfully."
    }
    ```

### 3. `prompts.py`
- **System Prompt**: "You are an expert data analyst. Convert the provided JSON data into a flat CSV structure based on the user's instruction."

## Integration
- Update `app/services/chatbot/tools/tools.py` to import `get_csv_generation_tool`.

## Usage Examples

**Scenario 1: Metadata List**
User: "Get me contracts ending this year and generate a CSV."
1.  Agent calls `metadata_search(...)` -> Returns `[{'id': 1, 'date': '2025-12-01'}, {'id': 2...}]`.
2.  Agent calls `csv_generation_tool(data=[{'id': 1...}, ...], instruction="contracts ending this year")`.
3.  Tool returns `{ "csv_content": "..." }`.

**Scenario 2: Comparison**
User: "Generate a CSV comparing key differences of contract 555 and 556."
1.  Agent calls `read_contracts(ids=[555, 556])` -> Returns `[{'id': 555, 'body': '...'}, {'id': 556...}]`.
2.  Agent calls `csv_generation_tool(data=[{'body': '...'}...], instruction="compare key differences")`.
3.  Tool returns `{ "csv_content": "..." }`.

**Scenario 3: Mixed Sources (Search + File)**
User: "Compare contract 123 with search results for 'NDA' and export CSV"
1.  Agent calls `semantic_search(query="NDA")` -> Returns `search_results` (List).
2.  Agent calls `get_file_content_tool(file_ids=["123"])` -> Returns `file_content` (Dict).
3.  Agent calls `csv_generation_tool(data={"search_results": search_results, "specific_file": file_content}, instruction="Compare key terms")`.
4.  Tool returns `{ "csv_content": "..." }`.
