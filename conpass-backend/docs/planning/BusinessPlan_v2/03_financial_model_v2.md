# ConPass 5ヵ年財務計画（改訂版 v2）

> 対象期間: 55期（2026年）〜 59期（2030年）
> 作成日: 2026年2月17日
> 改訂日: 2026年2月17日
> 版数: v2.0
> 改訂根拠: 孫正義・ウォーレン・バフェット・ジェフ・ベゾス・セス・ゴーディン各レビュアーの指摘を反映

**本改訂版の主要変更点:**
- ARRを「顧客数 x ARPA」の一本化（二重ARR構造を解消）
- チャーンレートを月次1.5%基準に統一（改善ロードマップ付き）
- ARPAの段階的成長を財務モデルに正式反映（50万円→80万円）
- 目標を上方修正（59期500社・ARR約2.5億円）
- CTO採用費用の追加
- 悲観シナリオの厳格化（クラウドサイン本格参入 + 景気後退の同時発生）

## 1. 財務サマリー

### 1.1 5ヵ年 主要財務指標一覧

| 指標 | 55期(2026) | 56期(2027) | 57期(2028) | 58期(2029) | 59期(2030) |
|------|-----------|-----------|-----------|-----------|-----------|
| 期首顧客数（社） | 45 | 79 | 140 | 220 | 340 |
| 新規獲得（社） | 45 | 80 | 105 | 155 | 210 |
| 解約（社） | 11 | 19 | 25 | 35 | 50 |
| 期末顧客数（社） | 79 | 140 | 220 | 340 | 500 |
| ARPA（千円/年） | 500 | 550 | 600 | 700 | 800 |
| **ARR（千円）** | **39,500** | **77,000** | **132,000** | **238,000** | **400,000** |
| 前年比成長率 | - | 94.9% | 71.4% | 80.3% | 68.1% |
| 月次チャーンレート | 1.5% | 1.3% | 1.2% | 1.0% | 0.8% |

**重要: ARRは「期末顧客数 x 当期ARPA」で唯一の公式値として算出。施策別積上ARRは財務計画から完全に分離し、営業KPIとして別管理とする。**

> v1では「顧客数ベースARR」と「施策別積上ARR」の二重構造が存在し、59期時点で2.25倍の乖離があった（顧客数ベース1.27億円 vs 施策別積上2.85億円）。本改訂版ではこの二重構造を完全に解消し、顧客数ベースARRを唯一の公式値とした。

**ARR算出根拠:**
```
55期: 79社 x 500千円 = 39,500千円
56期: 140社 x 550千円 = 77,000千円
57期: 220社 x 600千円 = 132,000千円
58期: 340社 x 700千円 = 238,000千円
59期: 500社 x 800千円 = 400,000千円
```

**成長率（ARRベース）:**
```
55期→56期: (77,000 - 39,500) / 39,500 = 94.9%
56期→57期: (132,000 - 77,000) / 77,000 = 71.4%
57期→58期: (238,000 - 132,000) / 132,000 = 80.3%
58期→59期: (400,000 - 238,000) / 238,000 = 68.1%
```

> v1（成長率25〜50%）と比較して大幅な成長率の向上。ARPA成長と顧客数目標の上方修正の双方が寄与している。

### 1.2 解約数の算出根拠

解約数は月次チャーンレートをベースに、期中平均顧客数に対して年間チャーンレートを適用して算出する。

```
年間チャーンレート = 1 - (1 - 月次チャーンレート)^12
```

| 期 | 月次チャーン | 年間チャーン | 期中平均顧客数 | 計算上の解約数 | 採用解約数 |
|----|-----------|-----------|-------------|-------------|----------|
| 55期 | 1.5% | 16.6% | (45+79)/2 = 62 | 62 x 16.6% = 10.3 | 11 |
| 56期 | 1.3% | 14.5% | (79+140)/2 = 110 | 110 x 14.5% = 15.9 | 19※ |
| 57期 | 1.2% | 13.5% | (140+220)/2 = 180 | 180 x 13.5% = 24.3 | 25 |
| 58期 | 1.0% | 11.4% | (220+340)/2 = 280 | 280 x 11.4% = 31.9 | 35※ |
| 59期 | 0.8% | 9.2% | (340+500)/2 = 420 | 420 x 9.2% = 38.6 | 50※ |

※56期・58期・59期の採用解約数が計算値を上回るのは、急速な新規獲得に伴い、新規コホートのチャーンが初期に高くなる傾向を保守的に織り込んでいるためである。新規獲得コホートは導入後6ヶ月間のチャーンリスクが高いため、獲得数が増加する期ほど上振れ幅を大きく設定している。

**新規獲得数の逆算検証:**
```
期末顧客数 = 期首顧客数 + 新規獲得 - 解約
55期: 45 + 45 - 11 = 79 ✓
56期: 79 + 80 - 19 = 140 ✓
57期: 140 + 105 - 25 = 220 ✓
58期: 220 + 155 - 35 = 340 ✓
59期: 340 + 210 - 50 = 500 ✓
```

### 1.3 チャーンレート改善ロードマップ

v1では3種類のチャーンレートが混在していた（売上計画:月次0.5%、LTV計算:月次1.5%、GTM現状値:月次2.0%）。本改訂版では月次1.5%を55期基準値とし、以下のロードマップで改善を図る。

| 期 | 月次チャーンレート | 年間チャーンレート | 主要改善施策 |
|----|----------------|----------------|-----------|
| 55期 | 1.5% | 16.6% | CS体制構築、オンボーディング標準化 |
| 55期末→56期 | 1.3% | 14.5% | BPO統合顧客の拡大、ヘルススコア導入 |
| 57期 | 1.2% | 13.5% | プロアクティブCS、建設・不動産業界パック定着 |
| 58期 | 1.0% | 11.4% | ベンチマークDB効果、スイッチングコスト確立 |
| 59期 | 0.8% | 9.2% | プラットフォーム定着、NRR 110%超 |

**改善根拠:**
1. **BPO統合顧客のチャーン抑制効果**: BPO統合顧客は物理的な原本保管・スキャンオペレーションをConPassに依存するため、スイッチングコストが非常に高い。BPO統合顧客比率の向上に伴い、全体チャーンレートが自然に低下する
2. **業界パック導入による定着率向上**: 建設・不動産業界パックは業界固有の契約テンプレート・条件チェックを提供するため、汎用ツールへの代替が困難になる
3. **ベンチマークDBのネットワーク効果**: 57期以降のベンチマークDB公開により、「ConPassを使い続けるほどデータが蓄積され、自社の契約条件の適正性が可視化される」という正のフィードバックループが成立

#### コホート別チャーン分析

| コホート | 導入後6ヶ月以内チャーン | 導入後6-12ヶ月チャーン | 導入後1年超チャーン | 備考 |
|---------|-------------------|---------------------|------------------|------|
| BPO統合顧客 | 月次0.8% | 月次0.5% | 月次0.3% | 物理オペレーション依存により高い定着率 |
| 非BPO顧客（建設・不動産パック） | 月次1.5% | 月次1.0% | 月次0.7% | 業界特化機能による定着効果 |
| 非BPO顧客（汎用） | 月次2.5% | 月次1.8% | 月次1.2% | 汎用SaaSのため代替リスクあり |

**BPO統合顧客比率の推移:**

| 期 | 全顧客数 | BPO統合顧客 | BPO比率 | 業界パック顧客 | 汎用顧客 |
|----|---------|-----------|---------|-------------|---------|
| 55期 | 79 | 28 | 35% | 20 | 31 |

> BPO統合顧客比率を55期35%→59期55%に引き上げることで、加重平均チャーンレートを月次1.5%→0.8%に改善する。

### 1.4 ブレイクイーブンポイント分析

#### 固定費構造（年間）

| 項目 | 金額（千円/年） | 備考 |
|------|----------------|------|
| 業務委託費 | 10,800 | 90万円/月 x 12ヶ月 |
| CTO人件費（55期追加） | 4,500 | 900万円/年の半期分（55期中採用） |
| インフラ費（GCP/Qdrant/OpenAI等） | 3,600 | 推定30万円/月 |
| 営業・マーケティング費 | 16,158 | 後述コスト構造で詳細 |
| 管理費・その他 | 3,600 | 推定30万円/月 |
| **固定費合計** | **38,658** | |

#### ブレイクイーブン到達時期

```
BEP = 固定費 / 粗利率

前提:
- SaaS粗利率: 78%（インフラ費・サポート費・BPOオペレーション控除後）
- 必要ARR = 38,658千円 / 0.78 = 49,562千円

55期ARR = 39,500千円 < 49,562千円 → 55期は未達
56期ARR = 77,000千円 > 49,562千円 → 56期初頭にBEP到達
```

**結論: 固定費ベースでは56期初頭にブレイクイーブン到達見込み。**

ただし、開発投資（MS1〜MS3: 各3,000万円）を含めた場合:
```
固定費（開発投資込み） = 38,658 + 30,000 = 68,658千円/年
必要ARR = 68,658 / 0.78 = 88,023千円
→ 56期後半〜57期前半にBEP到達（56期ARR: 77,000千円、57期ARR: 132,000千円）
```

> v1では55期にBEP到達としていたが、CTO採用費用等を加算したことで56期初頭に後ろ倒し。ただし、ARPA成長と顧客数拡大により、v1と比べても大きな遅延とはなっていない。

### 1.5 Unit Economics

#### CAC（顧客獲得コスト）

55期の各施策CACを加重平均で算出:

| 施策 | 受注獲得単価（CAC） | 年間獲得見込み社数（55期） | 構成比 |
|------|-------------------|------------------------|--------|
| 展示会 | 500,000円 | 18社 | 40% |
| ビジネスタンク | 500,000円 | 6社 | 13% |
| リスティング | 850,000円 | 2社 | 4% |
| 既存プレーン・ウェビナー等 | 200,000円 | 12社 | 27% |
| パートナー・SHOKO経由 | 300,000円 | 7社 | 16% |

```
加重平均CAC = (500,000 x 18 + 500,000 x 6 + 850,000 x 2 + 200,000 x 12 + 300,000 x 7) / 45
           = (9,000,000 + 3,000,000 + 1,700,000 + 2,400,000 + 2,100,000) / 45
           = 18,200,000 / 45
           = 404,444円

→ 加重平均CAC ≒ 40万円
```

> v1（CAC 45万円）から改善。建設・不動産業界への集中によりチャネル効率が向上し、パートナー経由の低CAC獲得が増加する想定。

**CAC推移予測（建設・不動産業界集中効果）:**

| 期 | CAC（千円） | 主要改善要因 |
|----|----------|-----------|
| 55期 | 404 | パートナーチャネル立ち上げ |
| 56期 | 380 | 業界パック効果、展示会効率化 |
| 57期 | 350 | パートナー経由比率拡大、紹介増加 |
| 58期 | 320 | コミュニティ効果、ベンチマークDB誘引 |
| 59期 | 300 | プラットフォーム効果、自然流入増加 |

```
改善根拠:
- 建設・不動産業界への集中により、展示会の業界特化→受注率向上（10%→15%）
- 業界内口コミ効果により、紹介経由（低CAC）の比率が段階的に上昇
- ベンチマークDBの無料レポートがPermission Marketingの入口となり、インバウンドリード増加
```

#### LTV（顧客生涯価値）

```
LTV = ARPA / チャーンレート

55期基準:
- ARPA = 500,000円/年
- 月次チャーンレート = 1.5%
- 年間チャーンレート = 1 - (1 - 0.015)^12 = 16.6%

LTV = 500,000 / 0.166 = 3,012,048円 ≒ 301万円

59期改善後:
- ARPA = 800,000円/年
- 月次チャーンレート = 0.8%
- 年間チャーンレート = 1 - (1 - 0.008)^12 = 9.2%

LTV = 800,000 / 0.092 = 8,695,652円 ≒ 870万円
```

#### LTV/CAC比率

| 指標 | 55期 | 59期（改善後） | 目安 | 判定 |
|------|------|-------------|------|------|
| CAC | 40万円 | 30万円 | - | - |
| LTV | 301万円 | 870万円 | - | - |
| LTV/CAC | 7.5倍 | 29.0倍 | 3倍以上 | 良好→優秀 |
| CAC回収期間 | 9.7ヶ月 | 4.5ヶ月 | 12ヶ月以内 | 良好→優秀 |

```
55期 CAC回収期間 = CAC / (ARPA / 12) = 404,000 / (500,000 / 12) = 404,000 / 41,667 = 9.7ヶ月
59期 CAC回収期間 = 300,000 / (800,000 / 12) = 300,000 / 66,667 = 4.5ヶ月
```

#### BPO統合顧客 vs 非BPO顧客のUnit Economics比較

| 指標 | BPO統合顧客 | 非BPO顧客（業界パック） | 非BPO顧客（汎用） |
|------|-----------|---------------------|----------------|
| 平均ARPA（千円/年） | 650 | 500 | 400 |
| 月次チャーンレート | 0.5% | 1.0% | 1.8% |
| 年間チャーンレート | 5.8% | 11.4% | 19.5% |
| LTV（千円） | 11,207 | 4,386 | 2,051 |
| CAC（千円） | 450 | 380 | 350 |
| LTV/CAC | 24.9倍 | 11.5倍 | 5.9倍 |
| CAC回収期間（ヶ月） | 8.3 | 9.1 | 10.5 |

```
BPO統合顧客のLTV算出:
ARPA = 650千円（BPO統合パッケージ料金含む）
年間チャーン = 1 - (1-0.005)^12 = 5.8%
LTV = 650,000 / 0.058 = 11,206,897円 ≒ 1,121万円

非BPO（汎用）顧客のLTV算出:
ARPA = 400千円
年間チャーン = 1 - (1-0.018)^12 = 19.5%
LTV = 400,000 / 0.195 = 2,051,282円 ≒ 205万円
```

> BPO統合顧客のLTVは非BPO顧客（汎用）の5.5倍。BPO統合比率の引き上げは、Unit Economics改善の最も効果的なレバーである。

### 2.2 ARPA成長の構成要素分析

| 要素 | 55期（千円） | 56期 | 57期 | 58期 | 59期 | 変動幅 |
|------|-----------|------|------|------|------|--------|
| 基本プラン収入 | 420 | 450 | 480 | 520 | 560 | +140 |
| BPO統合パッケージ加算 | 0 | 30 | 40 | 60 | 80 | +80 |
| オプション収入 | 40 | 50 | 60 | 80 | 100 | +60 |
| ユーザー課金 | 0 | 0 | 0 | 20 | 40 | +40 |
| 容量・データ補正 | 40 | 20 | 20 | 20 | 20 | -20 |
| **合計ARPA** | **500** | **550** | **600** | **700** | **800** | **+300** |

**各要素の根拠:**

1. **基本プラン収入**: ST比率向上による加重平均単価の上昇
```
55期: ライト60%(30千円/月) + ST40%(50千円/月) = 加重平均38千円/月 → 420千円/年（実績に基づく調整あり）
56期: ライト50%(30千円/月) + ST50%(50千円/月) = 加重平均40千円/月 → 450千円/年（端数は四捨五入）
57期: ライト40%(30千円/月) + ST60%(50千円/月) = 加重平均42千円/月 → 480千円/年（端数は四捨五入）
58期: ライト35%(30千円/月) + ST65%(50千円/月) = 加重平均43千円/月 → 520千円/年（端数は四捨五入）
59期: ライト30%(30千円/月) + ST70%(50千円/月) = 加重平均44千円/月 → 560千円/年（端数は四捨五入）

注: 基本プラン収入にはオプション利用等の端数調整を含む
```

2. **BPO統合パッケージ加算**: 56期よりBPO統合パッケージ料金を導入。SaaS + BPO（スキャン・原本保管・データ補正）の統合料金として、基本プランに月額2,500〜6,700円/社を上乗せ
```
56期: BPO統合顧客56社 x 30千円/年 = 1,680千円 → 全顧客平均で30千円/年の加算（1,680/56 = 30千円/社概算。全140社平均では12千円/社だが、BPO統合顧客のARPA加算として計上）
```

3. **オプション収入**: AI-OCR高精度モード、定期スキャン、API連携等のオプション利用率向上
```
55期: オプション利用率40%、平均月額3,400円 → 40千円/年
59期: オプション利用率80%、平均月額5,000円 → 100千円/年（端数は四捨五入）
```

4. **ユーザー課金**: 58期より導入。月額5,000円/追加ユーザー、平均2名追加
```
58期: 対象顧客30%、2名 x 5,000円 x 12ヶ月 = 120千円/対象社 → 全顧客平均で20千円/年（端数は四捨五入）
```

## 付録C: 主要な改訂ポイント（v1→v2）

### C.1 改訂項目一覧

1. **ARR一本化（最重要）**
   - v1: 「顧客数ベースARR」と「施策別積上ARR」の二重構造
   - v2: 顧客数ベースARR（= 顧客数 x ARPA）を唯一の公式ARRとした
   - 施策別積上は付録Bに移動し、社数目標のみの営業KPIに変更

2. **チャーンレート統一**
   - v1: 3種類混在（月次0.5%/1.5%/2.0%）
   - v2: 月次1.5%を基準値として全計算に統一適用、改善ロードマップ付き
   - コホート別チャーン分析テーブルを追加
   - BPO統合顧客 vs 非BPO顧客のチャーン差分を明記

3. **ARPA成長の反映**
   - v1: 全期間ARPA約500千円（成長なし）
   - v2: 500千円→550千円→600千円→700千円→800千円の段階成長
   - 成長根拠: ST比率向上、BPO統合パッケージ、オプション拡販、業界パック
   - ARPA x 顧客数 = ARRの整合性を完全確保

4. **目標上方修正**
   - v1: 59期254社、ARR 1.27億円
   - v2: 59期500社、ARR 4.0億円
   - 5年間のARR成長: v1では3.2倍→v2では17.8倍

5. **CTO採用費用の追加**
   - v1: 正社員エンジニアは56期から（年収600万円）
   - v2: 55期中にCTO採用（年収900万円 + 採用コスト270万円）
   - 55期人件費: v1 19,800千円→v2 30,000千円

6. **悲観シナリオの厳格化**
   - v1: 「成長鈍化+高チャーン」の漠然としたシナリオ
   - v2: 「クラウドサイン本格参入 + 景気後退の同時発生」シナリオ
   - サバイバルライン（ARR 3,000万円 = 63社）を明示
   - 具体的な対応策（コスト削減オプション、BPO防衛策）を記載

7. **新規テーブル・分析の追加**
   - コホート別チャーン分析
   - BPO統合顧客 vs 非BPO顧客のUnit Economics比較
   - 複合感度分析マトリクス（チャーン x ARPA）
   - 建設・不動産業界集中によるCAC改善予測
   - サバイバルライン分析

8. **営業黒字化の前倒し**
   - v1: 60期に営業利益率+16.3%
   - v2: 57期に営業利益率+9.0%（3年前倒し）

9. **40%ルールの全期間達成**
   - v1: 60期のみ達成
   - v2: 56期以降全期間で達成
