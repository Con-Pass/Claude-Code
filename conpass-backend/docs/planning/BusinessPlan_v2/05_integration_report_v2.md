### A.2 ARR

#### 各セクションの59期ARR記載値

| セクション | 59期ARR記載値 | 算出方法 |
|----------|-------------|---------|
| 01_market_research | 2.5億円（250,000千円） | SOM上方修正として記載 |
| 02_product_strategy | 2.5億円※（保守的計画値）/ 4.0億円（アップサイド） | 注記で二重記載 |
| 03_financial_model | **4.0億円（400,000千円）** | 500社 x 800千円 = 400,000千円 |
| 04_gtm_sales_strategy | **2.5億円（250,000千円）** | 冒頭テーブルで明記 |

#### 不整合の詳細

**最も重大な不整合**: 03_financial_model と 04_gtm_sales_strategy の59期ARRが一致しない。

- **03_financial_model_v2（セクション1.1）**: `59期: 500社 x 800千円 = 400,000千円`（ARR=4億円）
- **04_gtm_sales_strategy_v2（セクション0 冒頭テーブル）**: `59期: 500社 x 800千円 = 250,000千円（2.5億円）`

500社 x 800千円 = 400,000千円（4億円）が算術的に正しい。04_gtm_sales_strategyの「250,000千円（2.5億円）」は計算上の誤りか、チャーン・中途解約を考慮した実効ARRの可能性がある。

**02_product_strategyの注記**: 「59期ARRは保守的計画値。ARPA 80万円 x 500社 = 4.0億円がアップサイド目標。2.5億円はチャーン・中途解約を考慮した公式ARR」

この注記から推察すると、以下の2つのARR概念が混在している:

1. **理論ARR（期末ストック）**: 期末顧客数 x ARPA = 500社 x 80万円 = **4.0億円**
2. **実効ARR（チャーン考慮）**: 年度途中の解約・新規獲得タイミングを考慮した年間実現ARR = 約**2.5億円**

**問題点**: v1で最大の課題とされた「二重ARR問題」が、**形を変えて残存している**。v1では「顧客数ベースARR vs 施策別積上ARR」の二重構造だったが、v2では「理論ARR vs 実効ARR」の二重構造が発生している。

**修正推奨:**

1. 財務モデル（03）が定義する「ARR = 期末顧客数 x ARPA」を唯一の公式ARRとする
2. 59期公式ARR = 500社 x 800千円 = **400,000千円（4.0億円）** を全セクションで統一
3. 01/04の「2.5億円」は実効ARRであれば、別途「年間売上見込み」等の名称で明記し、公式ARRとは明確に区別する
4. 02の注記にある二重表記を解消し、公式ARR一本に統一する

**なお、各期のARR整合性は以下の通り:**

| 期 | 03_financial | 04_gtm | 02_product | 01_market | 判定 |
|----|------------|--------|-----------|----------|------|
| 55期 | 39,500千円 | 39,500千円 | 5,000万円（≒50,000千円） | 40,000千円 | **02不整合** |
| 56期 | 77,000千円 | 77,000千円 | 1.1億円（≒110,000千円） | 82,500千円 | **01/02不整合** |
| 57期 | 132,000千円 | 132,000千円 | 1.92億円（≒192,000千円） | 150,000千円 | **01/02不整合** |
| 58期 | 238,000千円 | 238,000千円 | 2.94億円（≒294,000千円） | 259,000千円 | **01/02不整合** |
| 59期 | 400,000千円 | 250,000千円 | 2.5億円/4.0億円 | 250,000千円 | **全セクション不整合** |

**02_product_strategyのARRは独自の顧客数（100/200/320/420/500社）x ARPAで算出されており、財務モデルと整合していない。** これは前述の顧客数不整合に起因する。

## 検証結論

ConPass 5ヵ年事業計画 改訂版（v2）は、v1で指摘された構造的欠陥の大部分を解消し、**戦略的一貫性と実行計画の具体性において大幅な改善**を達成した。特に以下の3点は高く評価できる:

1. **「1業界・1Moat・1ストーリー」の徹底**: 建設・不動産集中、BPO統合、「契約は、力だ。」のナラティブが全セクションに浸透
2. **顧客起点への転換**: 具体的なペルソナ、導入事例ストーリー、顧客視点メトリクスの導入
3. **財務モデルの精緻化**: コホート分析、感度分析、サバイバルライン分析の追加

一方で、**ARRの二重定義と02_productの顧客数不整合は、v1の「二重ARR問題」の変形として残存しており、早急な修正が必要**である。これらは算術的な統一で解消できる軽微な修正であり、計画の根幹を揺るがすものではない。

**最終判定: B+（良好） — 上記の残存課題を修正すれば、A-（優良）の水準に到達可能。**
