# ConPass 5ヵ年事業計画書 - プロダクト戦略

---

## 1. プロダクトビジョンと最終到達点

### 1.1 ビジョンステートメント

> **ConPassは「商取引インテリジェンス・プラットフォーム」として、契約管理の効率化を超え、商取引における情報格差を解消し、企業規模に関係なく健全な取引ができる社会を実現する。**

ConPassが目指すのは単なる契約管理SaaSではない。契約 → 取引 → 決済のトレーサビリティを一気通貫で提供し、日本の商取引における「信用インフラ」となることである。

現在の契約管理市場では、大企業は法務部門と顧問弁護士を擁し、有利な契約条件を勝ち取る一方、中小企業は情報・ノウハウの不足から不利な条件を受け入れざるを得ないという構造的な非対称性が存在する。ConPassはAIとデータの力でこの格差を埋め、すべての企業が対等な立場で商取引に臨める世界を創出する。

### 1.2 社会に届ける3つの価値

#### (1) 取引の非対称性解消

| 現状の課題 | ConPassが提供する解決策 |
|---|---|
| 大企業のみが業界標準の契約条件を把握 | ベンチマークDBにより中小企業も業界標準を参照可能 |
| 法務リソースの有無が交渉力を左右 | AIリスクチェック・交渉戦略AIが法務機能を民主化 |
| 過去の交渉履歴が属人的知識に依存 | ナレッジグラフが交渉履歴を組織知として蓄積・活用 |

ベンチマークDBは匿名化された契約条件の業界横断統計であり、中小企業が「業界ではこの条件が標準です」と根拠を持って交渉できるようになる。これは情報の非対称性を根本から覆す仕組みである。

#### (2) 商取引の健全性可視化

| 現状の課題 | ConPassが提供する解決策 |
|---|---|
| 契約と実際の取引実態の乖離が不可視 | 契約 → 証憑 → 決済の3点照合による自動検証 |
| 取引先の信用リスクが事後的にしか判明しない | 取引先信用情報連携（TDB/TSR）によるリアルタイム監視 |
| コンプライアンス違反の発見が遅延 | 定期クローリング＋プッシュ型アラートによる予防的監視 |

契約書を「結んで終わり」にしない。契約の履行状況を継続的にモニタリングし、取引の健全性をリアルタイムに可視化することで、不正や不履行を未然に防ぐ。

#### (3) 経営判断の民主化

| 現状の課題 | ConPassが提供する解決策 |
|---|---|
| 経営ダッシュボードは高額なBIツールが前提 | ConPassが契約データから経営指標を自動生成 |
| 業界動向の把握に専門コンサルが必要 | クロス業界ベンチマークによるセルフサービス分析 |
| 予測的リスク分析は大企業の専売特許 | AI予測モデルが中小企業にも予測的リスク分析を提供 |

契約データは経営の最上流に位置する情報である。ConPassはこのデータを構造化・分析し、専門家不在でも精度の高い経営判断ができる環境を提供する。

### 1.3 5つのMoat（競争優位の壁）戦略

ConPassの持続的競争優位は、以下の5つのMoatによって構築される。各Moatは単独でも価値があるが、相互に連携することで模倣困難な複合的優位性を形成する。

```
                    ┌─────────────────────────────┐
                    │    ベンチマークDB (Moat 1)    │
                    │  匿名化契約条件の業界横断統計  │
                    └──────────┬──────────────────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
┌─────────▼──────────┐ ┌──────▼──────────┐ ┌───────▼─────────┐
│ ナレッジグラフ      │ │ 規制準拠認証    │ │ BPO統合         │
│ (Moat 2)           │ │ (Moat 3)       │ │ (Moat 4)       │
│ 契約本文×法令×判例  │ │ 電帳法・ISMAP   │ │ スキャン→OCR→   │
│ ×証憑×交渉履歴     │ │ SOC2等         │ │ AI解析→保管→廃棄│
└─────────┬──────────┘ └───────┬────────┘ └───────┬─────────┘
          │                    │                    │
          └────────────────────┼────────────────────┘
                               │
                    ┌──────────▼──────────────────┐
                    │  業界特化ナレッジ (Moat 5)   │
                    │ テンプレート・検査ルール・商慣習│
                    └─────────────────────────────┘
```

| Moat | 構築開始 | 競争優位の本質 | 模倣困難性 |
|---|---|---|---|
| **1. ベンチマークDB** | MS2（匿名化パイプライン） | ネットワーク効果：利用企業が増えるほどデータの価値が向上 | データ蓄積に時間を要し、後発は追いつけない |
| **2. ナレッジグラフ** | MS1（RAG基盤改善） | 知識の構造化：契約本文×法令×判例×証憑×交渉履歴の多次元連携 | 9つのナレッジソースの統合は技術的に高難度 |
| **3. 規制準拠認証** | MS4（ISMAP/SOC2） | 参入障壁：認証取得コストと期間が競合の市場参入を阻む | 認証取得には1年以上の準備と数千万の投資が必要 |
| **4. BPO統合** | MS0（既存サービス） | オペレーション優位：日本パープルの既存BPO網を活用 | 物理的なBPOインフラの構築は新規参入者には困難 |
| **5. 業界特化ナレッジ** | MS1（足場業界パック） | ドメイン知識：業界固有の商慣習・検査ルールの蓄積 | 業界ごとの深い知識蓄積に専門家協力と時間が必要 |

**Moat間のシナジー：** ベンチマークDB（Moat 1）のデータはナレッジグラフ（Moat 2）の推論精度を高め、業界特化ナレッジ（Moat 5）はベンチマークDBのセグメント分析を可能にする。BPO統合（Moat 4）は紙契約のデジタル化を通じてデータ量を増加させ、規制準拠認証（Moat 3）はエンタープライズ顧客の獲得を加速し、さらなるデータ蓄積を促進する。この循環構造こそがConPassの本質的な競争優位である。

### 2.3 課題と制約 - ST比率10%の原因分析

現在のスタンダード（ST）プラン比率は約10%にとどまっている。この低さは売上成長とARPA向上の最大のボトルネックである。

```
ST比率10%の原因構造:

┌──────────────────────────────────────────────────────────┐
│                ST比率 10% （目標 70%+）                    │
├──────────────┬──────────────┬─────────────┬──────────────┤
│  機能差別化   │  体験品質    │  営業構造   │  価格認知    │
│  の不足      │  の課題      │  の課題     │  の課題      │
├──────────────┼──────────────┼─────────────┼──────────────┤
│・AIアシスタント│・Word乖離の │・ライトから │・ST月額5万の │
│ がプル型のみ  │  エディタ   │ のアップ   │  価値が不明確│
│・WF条件分岐  │・電子契約   │  セル導線  │・AIオプション│
│  不可        │  連携断絶   │  が弱い    │  の価値訴求  │
│・RAG知識基盤 │             │            │  不足       │
│  が貧弱      │             │            │             │
└──────────────┴──────────────┴─────────────┴──────────────┘
```

**根本原因の分析：**

1. **機能差別化の不足：** STプランでしか使えない「圧倒的な価値」がない。AIアシスタントはプル型のみで、ユーザーが質問しなければ価値を体感できない。ワークフローの条件分岐不可は実務上の重大な制約であり、STプランの魅力を損なっている。

2. **体験品質の課題：** TinyMCEエディタはWord実務との乖離が大きく、ユーザーが「使いにくい」と感じる最大の要因。電子契約連携（GMO Sign）の断絶は、契約業務の一気通貫を阻害し、他ツールとの併用を余儀なくしている。

3. **営業構造の課題：** ライトプランからSTプランへのアップセル導線が体系化されていない。ライトプランで十分と感じるユーザーに対して、STプランの追加価値を示すタッチポイントが設計されていない。

4. **価格認知の課題：** STプラン月額50,000円の費用対効果が定量的に示されておらず、AIオプション15,000円/月の価値訴求も不足している。

**プッシュ型AIへの転換が意味するもの：**

```
【MS1まで：プル型AI】              【MS2以降：プッシュ型AI】

ユーザー → 質問 → AI → 回答        AI → 検知 → アラート → ユーザー

・ユーザーが質問しないと          ・契約更新期限の自動通知
  価値を発揮しない               ・法改正による影響契約の特定
・AIの存在を忘れられる            ・取引先信用変動の即時警告
                                  ・コンプライアンス違反の予防検知
```

この転換により、AIの価値がユーザーの能動的行動に依存しなくなり、ST/AIオプションの価値訴求が格段に容易になる。

```

### 4.2 マルチエージェント戦略

ConPassのAIは、単一のモノリシックなAIではなく、専門化されたエージェント群が協調する「マルチエージェント・アーキテクチャ」を採用する。

**エージェント構成の進化：**

```
【MS0-MS1：基盤エージェント群】
┌──────────┐  ┌──────────┐  ┌──────────┐
│ OCRエージェント│ │ 台帳エージェント│ │ QAエージェント │
│ (文書解析)    │ │ (メタデータ抽出)│ │ (質問応答)    │
└──────────┘  └──────────┘  └──────────┘
        ▼              ▼              ▼
【MS2-MS3：監視・分析エージェント群追加】
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 監視エージェント│ │ リスクエージェント│ │ ドラフトエージェント│
│ (期限/法改正) │ │ (リスク評価)    │ │ (契約書起草)    │
└──────────┘  └──────────┘  └──────────┘
        ▼              ▼              ▼
【MS4-MS5：戦略・予測エージェント群追加】
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 交渉エージェント│ │ 予測エージェント│ │ 経営分析エージェント│
│ (交渉戦略提案) │ │ (リスク予測)   │ │ (経営指標分析)    │
└──────────┘  └──────────┘  └──────────┘
```

**エージェント間の協調パターン：**

- **シーケンシャル：** OCRエージェント → 台帳エージェント → リスクエージェント（契約書アップロード時の一連処理）
- **パラレル：** 監視エージェントが複数の契約を同時に定期チェック
- **ルーティング：** ユーザーの質問内容に応じて最適なエージェントにルーティング（MS2のAIルーティングWFで実現）
- **合議制：** 交渉エージェントが、ベンチマークDB参照エージェント + 判例参照エージェント + リスクエージェントの結果を統合して戦略提案（MS4）

| AIアシスタント（オプション） | +15,000円 | - | AI質問応答 |
| API連携（オプション） | +10,000円 | - | 外部システム連携 |

**現行料金体系の課題：**

1. **ライトとSTの価格差が小さい（20,000円/月）：** にもかかわらずST比率が10%にとどまっているということは、20,000円分の追加価値がユーザーに伝わっていない。
2. **AIオプションが別課金：** AI機能がConPassの中核価値であるにもかかわらず、オプション扱いにより採用率が低い可能性がある。
3. **ユーザー数に依存しない定額制：** 大企業ほど割安になる反面、中小企業にとっては割高感がある。ARPA向上の天井にもなる。
4. **エンタープライズプランの不在：** 大企業向けの提案ができず、ARPA 50万円以上の案件を逃している。

### 6.2 ARPA 50万 → 100万への引き上げ戦略

**料金戦略の進化ロードマップ：**

#### Phase 1：ST価値強化期（MS1-MS2）

```
現行                          Phase 1 改定案
─────────────                 ─────────────────────
ライト:  30,000円/月           ライト:     30,000円/月（据置）
ST:      50,000円/月           スタンダード: 60,000円/月（AI基本機能込み）
AIオプション: +15,000円/月      AIプロ:      +20,000円/月（プッシュ型AI）
API連携:  +10,000円/月          API連携:     +15,000円/月

想定ARPA: ~40,000円            想定ARPA: ~55,000円
```

**改定のポイント：**
- STプランにAI基本機能（プル型QA、リスクチェック）を標準搭載し、価格を60,000円に改定
- AIオプションを「AIプロ」に改称し、プッシュ型監視・レポート自動生成等の高度機能を提供
- STプランの価値を明確化し、アップセル率向上を狙う

#### Phase 2：ユーザー課金導入期（MS3-MS4）

```
Phase 2 改定案
─────────────────────────────────────────────
スターター:     30,000円/月 + 3,000円/ユーザー（上限5名）
ビジネス:       60,000円/月 + 5,000円/ユーザー（上限20名）
ビジネスAI:     80,000円/月 + 7,000円/ユーザー（上限50名）
エンタープライズ: 200,000円/月〜 + 9,000円/ユーザー（無制限）

想定ARPA: ~100,000円（ビジネスプラン10ユーザーの場合: 110,000円）
```

#### Phase 3：プラットフォーム課金期（MS5）

API Business:           +100,000円/月
テンプレートマーケット:   従量課金（テンプレート単位）

想定ARPA: ~150,000円以上
```

### 6.3 エンタープライズプラン設計（MS4）

| 項目 | 内容 |
|---|---|
| **基本料金** | 200,000円/月〜（個別見積） |
| **ユーザー課金** | 9,000円/ユーザー/月 |
| **初期費用** | 500,000円〜（導入支援込み） |
| **SLA** | 99.9%稼働率保証 |
| **セキュリティ** | ISMAP / SOC2 Type II準拠 |
| **サポート** | 専任CSM、優先サポート |
| **カスタマイズ** | 専用WFテンプレート、API個別開発 |
| **契約期間** | 年間契約（月額10%割引） |

**エンタープライズプランの想定ARPA：**

- 50名利用の場合：200,000 + 9,000 x 50 = 650,000円/月（年間780万円）
- 100名利用の場合：200,000 + 9,000 x 100 = 1,100,000円/月（年間1,320万円）
- 200名利用の場合：個別見積（年間2,000万円〜）

### 6.4 ユーザー課金モデルの可能性

**ユーザー課金（月額9,000円/ユーザー）の根拠：**

| 比較対象 | ユーザー単価 | ConPass優位性 |
|---|---|---|
| クラウドサイン | 10,000円/送信件数ベース | 月額定額で予算管理が容易 |
| Hubble | 要問合せ（高価格帯） | 中小企業にも手が届く価格設定 |
| LegalForce | 要問合せ（高価格帯） | AI＋BPO統合で総コスト削減 |

**ユーザー課金のメリット：**
- 利用規模に応じた公平な課金
- 大企業からのARPA向上
- 利用者数の可視化によるエンゲージメント把握

**ユーザー課金のリスクと対策：**
- リスク：ユーザー数を絞られる（シートの共有利用）
- 対策：監査ログ・個人別アクティビティにより個別アカウントの必要性を訴求
