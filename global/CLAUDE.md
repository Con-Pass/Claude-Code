# 基本設定
- 日本語で回答してください
- コミットメッセージも日本語で書いてください

# ワークフロー
- コード変更後は必ず `npx tsc --noEmit` で型チェックを実行
- コミット前にlintとtypecheck両方を通す
- 大きな変更はPRを作ってからマージ

# コード規約
- TypeScriptを使用する場合はstrict modeで型安全を保つ
- `any` 型は使わない
- エラーハンドリングは必ずする（unhandled promiseを作らない）

# 姿勢
- 過剰実装しない。求められた最小限の変更のみ行う
- 既存コードを読んでから変更提案する
- セキュリティ上のリスク（SQLi, XSS, 秘密情報露出など）は即指摘する
